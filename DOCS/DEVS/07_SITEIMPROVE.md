# SiteImprove Monitoring Workflow

This document provides a general overview of how SiteImprove monitoring works and how issues are reviewed and tracked.

<br />

### SiteImprove Overview

SiteImprove scans the website and evaluates several categories of site quality.
These categories include:

* Digital Certainty Index (overall site health)
* Quality Assurance (broken links, readability, outdated content)
* Accessibility (WCAG compliance)
* SEO (search engine optimization)
* Policy and compliance checks

Each category contributes to an overall score displayed in the SiteImprove dashboard.

<br />

### Requesting or Running Site Crawls

SiteImprove runs monthly crawls to detect issues across pages and documents. A manual crawl can also be run at the bottom of the DCI score dashboard.

During a crawl, SiteImprove scans:

* Web pages
* Links and navigation
* Images and alt text
* Heading structures
* Accessibility attributes
* PDFs and downloadable files

The crawl results are used to generate reports and update the SiteImprove score dashboard.

<br />

### Reviewing SiteImprove Reports

You can review reports in the SiteImprove dashboard.

1. Open the SiteImprove dashboard.
2. Review the **Digital Certainty Index (DCI)** summary.
3. Check individual categories such as:
   * Quality Assurance
   * Accessibility
   * SEO
4. Review flagged issues and affected pages.

The dashboard also provides a breakdown of issues by type and occurrence.

<br />

### Accessibility Monitoring

SiteImprove identifies accessibility violations based on WCAG standards.

These violations are categorized by W3C classification levels:

* Level A
* Level AA
* Level AAA

Common accessibility issues include:

* Contrast violations
* Missing alternative text for images
* Improper heading structure
* Focus visibility issues
* Link purpose clarity
* Interactive element sizing

Accessibility issues can be reviewed under the **Accessibility section** of the SiteImprove dashboard.
